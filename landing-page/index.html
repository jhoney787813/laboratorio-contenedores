2. Paso a Paso de la Construcción

  1. Diseño de la Arquitectura y DDD: Se modelaron los agregados de dominio (Usuario y Mensaje) y se definió el uso de Redis para almacenar más de 15 tipos de datos distintos mediante Hashes y Listas.
  2. Desarrollo del Servidor de Chat: Se construyó una aplicación con Express y Socket.io para manejar conexiones entrantes y generar salas privadas para cada usuario.
  3. Creación del Bot Autónomo: Se implementó un cliente Socket.io en un proceso Node.js separado que responde automáticamente a palabras clave en las salas privadas.
  4. Implementación del Panel de Administración: Se construyó un microservicio para visualizar la base de datos en tiempo real y enviar alertas mediante Redis Pub/Sub.
  5. Contenerización: Se escribieron los Dockerfile para cada servicio Node.js y se orquestaron utilizando docker-compose.yml asegurando la persistencia a través de volúmenes compartidos.

3. Pruebas y Validación del Sistema

Sigue estos pasos para probar la integridad de la aplicación de software:

  • Levantar el entorno: Ejecuta docker-compose up -d --build en la raíz del proyecto.
  • Prueba del Cliente: Ingresa a http://localhost:3000, escribe tu nombre y envía un mensaje como "hola" o "docker". El Bot debería responder en unos segundos.
  • Prueba de Aislamiento: Abre una ventana de incógnito, entra al chat con otro nombre. Verifica que los historiales son completamente independientes (chats 1-a-1).
  • Prueba de Administración y Pub/Sub: Abre http://localhost:4000. Envía una alerta global. Verifica en las ventanas del chat que la alerta roja aparece instantáneamente, validando la comunicación asíncrona a través de Redis.
  • Auditoría de BD: Ingresa al contenedor de Redis (podman exec -it chat_redis redis-cli) y usa el comando HGETALL user:ID para validar la integridad y tipos de los datos persistidos.
