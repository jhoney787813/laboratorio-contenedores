2. Justificación: Arquitectura Moderna Orientada a Microservicios

El diseño de esta solución responde a los estándares actuales de la industria para arquitecturas escalables y desacopladas:

  • Desacoplamiento mediante Eventos (Pub/Sub): En lugar de que el servidor de administración se comunique directamente con el servidor de chat mediante HTTP (lo cual genera un acoplamiento fuerte y latencia), se utiliza el patrón de diseño Publish/Subscribe nativo de Redis. Esto significa que los microservicios son agnósticos unos de otros; el administrador simplemente dispara un evento global, garantizando alta cohesión y bajo acoplamiento.
  • Comunicación Asíncrona (WebSockets): Las aplicaciones modernas en tiempo real evitan el "polling" (consultas repetitivas al servidor). Al usar Socket.io sobre Node.js, mantenemos conexiones TCP persistentes y bidireccionales de baja latencia.
  • Contenerización Aislada: Cada componente (Chat, Bot, Admin, DB) posee su propio `Dockerfile`. Esto asegura el principio de responsabilidad única. Si el Chat Bot falla, no afecta al Chat Server; si necesitamos escalar el servidor web, podemos instanciar réplicas sin tocar la lógica de la base de datos o del panel de administración.
  • Single Source of Truth (SSOT): Redis actúa como la única fuente de verdad rápida en memoria. Mantiene un almacenamiento de estado centralizado, permitiendo que las aplicaciones servidoras (Node.js) se mantengan Stateless (sin estado interno). Esto facilita escalar horizontalmente en la nube de ser necesario.

4. Instrucciones de Ejecución (Basado en el README Oficial)

El siguiente es el extracto de la documentación principal para levantar y auditar el sistema en la máquina local o servidor.

# 1. Construir y Levantar el Entorno (Orquestación con Podman/Docker)
docker-compose up -d --build

# 2. Uso de la Aplicación
• Abre tu navegador en: http://localhost:3000 (Chat Web del Usuario).
• Abre el Panel Admin en: http://localhost:4000 (Envío de alertas masivas).

# 3. Auditoría de Datos en Base de Datos (Validación de Tipos)
# Entrar a la terminal interactiva de la base de datos:
podman exec -it chat_redis redis-cli

# Mostrar a todos los usuarios autenticados:
SMEMBERS active_users

# Consultar propiedades persistidas de un usuario (Ejemplo: IP, Booleanos, Strings):
HGETALL "user:"

# Revisar el historial asíncrono y los registros JSON en Listas:
LRANGE "chat:messages:" 0 -1
